## When this job is hot

The user knows certain apps are easy to open before they have done the thing they meant to do. This can be self-authored, like an adult or responsible teen who wants help building better patterns, or supported by someone else, like a parent who wants a child to set goals or manage to-dos before entertainment apps are available.

## What "done" feels like for the user

There is a calm, chosen pause between impulse and drift. The user does one real Kwilt action first, then the restriction gets out of the way. The guardrail feels like support for agency, not punishment or surveillance.

## Notes

This job is broader than Focus Sessions. Focus Protection protects a session the user already started; this job protects the moment before low-intention app use. It must stay optional, reversible, and non-shaming.

For teen and family use, there are two different patterns:

- A responsible teen self-installs because they want better phone patterns.
- A parent or guardian helps set the rule because they want Kwilt work to happen before entertainment apps.

In both cases, the product still needs to preserve trust for the person whose apps are restricted: clear setup, visible rules, no hidden monitoring, and simple ways to understand what unlocks access.
