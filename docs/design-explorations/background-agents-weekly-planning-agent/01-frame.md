# Frame: background-agents-weekly-planning-agent

## What the user said

> Kwilt should explore Background Agents / Weekly Planning Agent as a Pro pillar, connected to Chapters but not collapsed into Chapters. Claude/ChatGPT are useful when the user is actively chatting; Kwilt can run rituals in the background while the user is asleep or away. Chapters remain retrospective. Agents propose; users confirm.

## Restated in user voice

When a week has ended and I have lived enough real life for patterns to emerge, I want Kwilt to quietly prepare a few grounded next steps for the week ahead, so I can re-enter the few things that matter without turning reflection into planning homework or trusting an AI to silently reshape my life.

## Anchors served

- `jtbd-move-the-few-things-that-matter` — the agent's output is not another recap; it proposes Activities and, when appropriate, Goals that help existing Arcs move in the coming week.
- `jtbd-make-sense-of-the-season` — Chapters remain the retrospective meaning surface, and the weekly agent can draw on that meaning without changing what a Chapter is.
- `jtbd-recover-when-i-drift-from-an-arc` — the weekly boundary is a humane moment to notice quiet Arcs and offer re-entry while avoiding shame or urgency.
- `jtbd-trust-this-app-with-my-life` — background automation must be transparent, reversible, and confirmed by the user; this is where the trust bar is highest.

```yaml
serves:
  - jtbd-move-the-few-things-that-matter
  - jtbd-make-sense-of-the-season
  - jtbd-recover-when-i-drift-from-an-arc
  - jtbd-trust-this-app-with-my-life
```

## Friction we're addressing

The connector makes active chat surfaces better at reading and writing Kwilt data, but it does not solve the moments when the user is not actively chatting. Weekly reflection already creates meaning; the missing bridge is a calm, confirmable next-step ritual that uses that meaning to help the user re-enter action.

## Out of scope

- No proactive push into old Claude/ChatGPT threads.
- No Chapter-generated future plans; Chapters stay retrospective only.
- No silent Goal, Arc, Force, or Activity anchoring changes.
- No dashboards, productivity scoring, or "optimize your week" framing.
- No general-purpose agent platform in the launch feature brief.

## Open question

Should the first launch be Pro-only from the first ritual run, or should free users receive a one-time preview proposal after their first Chapter to demonstrate the Pro value?
