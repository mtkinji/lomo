# Yes-And: To-Do Organization Triage

Original idea: Treat a customer's "I have so many to-dos in Kwilt; I need a better way to organize them" as a design sprint, because the existing views system does not fully answer the need.

**Yes, and what if it could...** notice when the user's Activity list has become noisy and offer a calm organizing pass.

- Serves: `jtbd-move-the-few-things-that-matter`
- Job elevation: Kwilt moves from showing more list controls to helping the user regain a sense of what matters.
- New value: The product can respond to overload before the user has to know that views exist.
- Cost delta vs. original: medium
- Anti-pattern check: pass, if the offer is quiet and never frames the user as behind.

**Yes, and what if it could...** create a temporary "triage" surface before it creates durable saved views.

- Serves: `jtbd-carry-intentions-into-action`
- Job elevation: The user can sort the pile into next actions, later items, waiting items, and maybe-not items without committing to a permanent system.
- New value: Organization becomes a recovery ritual, not another configuration burden.
- Cost delta vs. original: medium
- Anti-pattern check: pass, if triage does not block capture or require every Activity to be classified.

**Yes, and what if it could...** recommend a few starter buckets based on Kwilt's object model instead of generic productivity categories.

- Serves: `jtbd-move-the-few-things-that-matter`
- Job elevation: Buckets reflect the user's Goals, scheduled work, unanchored capture, and stale commitments rather than arbitrary labels.
- New value: The user sees why their to-dos exist in the first place.
- Cost delta vs. original: medium
- Anti-pattern check: pass, if AI suggestions are previewed and editable rather than silently applied.

**Yes, and what if it could...** let the user finish triage by choosing one small next move.

- Serves: `jtbd-carry-intentions-into-action`
- Job elevation: Organization ends in action, not in a prettier backlog.
- New value: The sprint connects directly to Kwilt's "what now?" spine and Activity Detail next-best-action work.
- Cost delta vs. original: low
- Anti-pattern check: pass, if this remains an invitation rather than a forced commitment.

**Yes, and what if it could...** turn the output into saved views only when the user wants the organization to persist.

- Serves: `jtbd-trust-this-app-with-my-life`
- Job elevation: Kwilt earns trust by keeping changes inspectable and reversible.
- New value: Saved views become the durable result of a job-serving ritual, not the starting point the user has to discover.
- Cost delta vs. original: medium
- Anti-pattern check: pass, if the user previews names, filters, and membership before saving.

**Yes, and what if it could...** expose "unanchored" and "stale" work as gentle observations.

- Serves: `jtbd-capture-and-find-meaning`
- Job elevation: Captured work can remain first-class while still revealing what needs a home, a schedule, or release.
- New value: Kwilt helps the user understand the shape of their life system without turning it into an audit.
- Cost delta vs. original: medium
- Anti-pattern check: pass, if language treats drift and unanchored capture as information.

**Frame Recommendation:** Run design-thinking-loop with an expanded frame. The job is not "make views easier to discover." It is "help an overloaded user re-enter a crowded Activity system and leave with fewer open loops, clearer buckets, and one honest next action."
