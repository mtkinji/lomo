# Yes-And: object-creation-ux-system

## Expansion

The offered idea is larger than FTUX. FTUX is the first exposure to a broader creation grammar:

- Users can start with concrete language.
- Kwilt identifies whether the user is naming a focus, a Goal, an Arc, or a reason.
- The flow asks only the next question needed to shape the right object.
- The saved result makes the object hierarchy feel obvious after the fact.

## Adjacent opportunities

- Reuse the same deterministic option families across FTUX, direct Goal creation, and direct Arc creation.
- Let direct Goal creation ask one extra identity/context question only when the Goal lacks an Arc or meaning signal.
- Let direct Arc creation support both identity-first and activity-to-identity entry paths.
- Add a creation-quality checklist that tests whether an Arc is identity-shaped and a Goal is concrete.
- Store the question grammar in one canonical package so copy, labels, hidden generation meanings, and tests do not drift.
- Use onboarding observations as pressure tests for normal creation flows.

## Job-elevation note

This is not just "better survey questions." It is about helping users trust Kwilt as the place where fuzzy intent becomes the right kind of life object.

## Frame recommendation

Run the loop with the expanded frame: a Creation UX System initiative with FTUX Goal+Arc as its first learning slice.
