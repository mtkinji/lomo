# Yes-And: todo-linked-goal-redundancy

## Expansion decision
Skip expansion.

## Rationale
This is a bounded refinement to an existing detail surface. It does not introduce a new capability, model, platform, AI behavior, or workflow. The bigger opportunity is not another metadata surface; it is making the existing To-do detail carry orientation and editing duties without duplication.

## Job-elevation note
The elevated job is not "show less text." It is "make the relationship between To-do and Goal obvious without making the user maintain the same relationship twice."

## Frame recommendation
Skip the broad loop and ship the original refinement inside the current Activity detail system.
