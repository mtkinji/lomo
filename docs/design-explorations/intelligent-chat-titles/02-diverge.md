# Diverge: Intelligent Chat Titles

Axis: deterministic stability versus semantic understanding.

## A. First-message extraction
Clean and truncate the opening user message. Fast and deterministic, but questions like “Can you help me think this through?” remain vague and never improve as the conversation becomes clearer.

## B. Lifecycle-aware model naming
Suggest a short name from the opening exchange, then refresh it only when the existing compression process forms a new durable summary. Protect manual names. This best matches how conversation understanding develops without making titles visibly churn.

## C. Retitle after every turn
Continuously ask the model for the current topic. It adapts quickly but wastes calls, makes the list unstable, and risks replacing a useful recognizable label because of one tangent.

All options affect only Chat continuity, preserve the four-object model and capture-first behavior, and introduce no dashboard, score, pressure, or anthropomorphic voice.
