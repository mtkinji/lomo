Generated by `npm run agent:map`. Keep this file committed so coding agents can orient quickly without rediscovering the repo shape every turn.

## Verification Commands

- `npm run verify:changed` - `node scripts/verify-changed.mjs`
- `npm run code:health` - `node scripts/code-health.mjs`
- `npm run test:code-health` - `node --test scripts/code-health-lib.test.mjs`
- `npm run lint` - `tsc --noEmit`
- `npm run lint:tests` - `tsc --noEmit --project tsconfig.test.json`
- `npm run test` - `jest`
- `npm run test:ci` - `jest --ci --coverage`
- `npm run lint:supabase-functions` - `find supabase/functions -path 'supabase/functions/node_modules' -prune -o -name '*.ts' ! -path '*/__tests__/*' -print0 | xargs -0 deno check --no-lock --config supabase/functions/tsconfig.json`
- `npm run product:lint` - `node scripts/product-lint.mjs`
- `npm run architecture:lint` - `node scripts/architecture-lint.mjs`
- `npm run mcp:smoke` - `node scripts/mcp-smoke.mjs`
- `npm run mcp:e2e` - `node scripts/mcp-all-tools-e2e.mjs`

## Completion Ritual

- Before handing work back, run `npm run verify:changed -- --run` so the current diff selects and executes the relevant local gates.
- `npm run code:health -- --fail-on-regression` protects the repo from new large-file, loose-typing, raw-Text, and direct-Reusables drift while leaving legacy debt warning-only.
- Add `npm test -- --runInBand` when shared config, shared stores/services, or test infrastructure changed.
- CI enforces the stable repo-wide gates: code health, app typecheck, test typecheck, Supabase function typecheck, product lint, architecture lint, and coverage Jest.

## Primary Code Areas

- `src/features/` - user-facing feature surfaces, each with a `FEATURE.md` manifest except dev-only tooling.
- `src/domain/` - pure domain types, workflow specs, limits, and transformation logic.
- `src/services/` - backend, AI, notification, analytics, sync, calendar, and integration clients.
- `src/store/` - Zustand stores and app state lifecycle logic.
- `src/ui/` - shared app UI adapters and layout primitives. Feature code should use this layer instead of raw Reusables.
- `supabase/functions/` - Deno Edge Functions, checked by `npm run lint:supabase-functions`.
- `packages/arc-survey`, `packages/kwilt-sdk`, `packages/kwilt-tokens` - workspace packages shared with app and connector surfaces.

## Agent Routing Heuristics

- Touching `supabase/functions/**` usually requires `npm run lint:supabase-functions`; tests under `_shared/__tests__` are Deno-oriented and outside the npm/Jest app gate.
- Touching `docs/jtbd`, `docs/personas`, `docs/job-flows`, `docs/feature-briefs`, or `src/features/*/FEATURE.md` requires `npm run product:lint`.
- Touching feature UI should run `npm run architecture:lint`; heed warnings when already editing files with legacy raw `Text` imports.
- Touching large screens should prefer extracting pure helpers/hooks with focused tests over broad inline edits.
- New code should stay inside the `code:health` ratchets; if a large-file or loose-typing exception is intentional, make the exception explicit in review instead of quietly expanding the baseline.
- Use `npm run verify:changed` at the start and end of an agent session to choose the right checks for the current diff.
