Target audiences, named representative personas, and JTBDs are peer demand-side anchors. Audiences name who we might serve, personas make that audience concrete, and JTBDs name what becomes active for that person in a situation. Each audience has a hero JTBD, with supporting and feature-level JTBDs below it. Job flows map those jobs to app flow, offerings, delivery scores, and gaps. They are used for roadmap prioritization, design-thinking, feature briefs, product review, analytics, growth positioning, and reflect-after-ship updates. feature briefs declare the primary `audiences:`, `personas:`, `hero_jtbd:`, `job_flow:`, and which JTBDs they `serves:`. See [`AGENTS.md`](../AGENTS.md) for the full solo-dev loop.
